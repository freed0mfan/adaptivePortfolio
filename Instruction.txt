AdaptivePortfolio — как запустить без терминала
=================================================

WINDOWS
-------
1. ВАЖНО: распакуйте архив целиком (правый клик по zip ->
   «Извлечь все...»). Запускать .bat прямо изнутри окна
   архиватора нельзя — он не увидит соседние файлы.
2. Зайдите в распакованную папку adaptive_portfolio.
3. Дважды кликните по файлу:  Запустить.bat
4. При первом запуске установка зависимостей займёт 5–10 минут.
   Дальше дашборд будет открываться за 5–10 секунд.
5. Браузер откроется автоматически по адресу http://localhost:8501

Если Windows ругается «Защитник Windows заблокировал…»:
- Нажмите «Подробнее» -> «Выполнить в любом случае».
  (Это нормально для неподписанных .bat-файлов.)

Если в окне видны странные символы вместо русских букв —
это косметика консоли, на работу не влияет: все служебные
сообщения внутри .bat написаны на английском.

Если на компьютере нет Python:
- Лаунчер сам откроет страницу загрузки.
- Установите Python 3.11 или новее, ОБЯЗАТЕЛЬНО отметив
  галочку «Add Python to PATH» на первом экране установщика.
- После установки снова дважды кликните Запустить.bat.


macOS
-----
1. Распакуйте архив.
2. Дважды кликните по файлу:  Запустить.command
3. При первом запуске macOS может сказать «не удаётся открыть,
   разработчик не определён». Тогда:
   - Кликните правой кнопкой по Запустить.command -> «Открыть»
   - В диалоге снова нажмите «Открыть».
4. Дальше всё автоматически.


ОСТАНОВКА
---------
Просто закройте окно терминала, которое открыл лаунчер.


ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК
-------------------------
Виртуальное окружение и лог хранятся НЕ в папке проекта,
а в:  %LOCALAPPDATA%\AdaptivePortfolio
(обычно это  C:\Users\<ваше имя>\AppData\Local\AdaptivePortfolio)

Чтобы открыть эту папку: Win+R -> вставьте
  %LOCALAPPDATA%\AdaptivePortfolio
и нажмите Enter.

Там лежат:
  venv\            — установленные библиотеки
  install.log      — подробный лог последнего запуска

Чтобы переустановить с нуля — удалите папку venv
и снова запустите Запустить.bat.

ПОЧЕМУ ТАК: если проект лежит в пути с русскими буквами
(например, "Курсовая 3 курс") или внутри Яндекс.Диска/OneDrive,
Python venv и pip могут ломаться из-за кодировки путей и
постоянной синхронизации файлов. Поэтому окружение создаётся
в отдельной ASCII-папке в AppData — это стабильнее.
